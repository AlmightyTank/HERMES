# HERMES 0.1.0-alpha14.0.8

Alpha 14.0.8 is the next request and server-analysis performance pass for SPT 4.0.13. It is based on Alpha 14.0.7.3 and also includes the first-load native HERMES header correction.

## Main changes

### Faster craft profitability preparation

Craft summary generation no longer asks the full trader and market detail services to construct display models for every unique craft output.

- Trader profit uses a lightweight best-base-item sale lookup.
- It does not scan trader purchase assortments or build item-detail purchase offers merely to learn the best trader sale value.
- Flea profit uses a lightweight base-item offer scan and fee-backed net estimate.
- Assembled offers with installed parts are excluded from this fast craft-output comparison.
- The existing per-summary output caches still ensure each unique output is valued only once.

The normal Items & Market and Traders screens keep their full detailed calculations.

### Complete Assistant feed preparation

`/hermes/assistant/prepare/...` now acts as the authoritative server join point for Hideout, Crafts, Stash, and Loadout materialization.

The server will not publish a prepared Assistant feed until all four source summaries are valid. This prevents a locally timed-out Crafts request from silently removing profitable-craft alerts from the first Assistant feed.

When a client timeout occurs but the server calculation later completes, the client rereads only the missing response from the completed server cache instead of leaving the initial snapshot incomplete.

### Workspace-open request coalescing

Repeated lifecycle callbacks generated by the same inventory-screen opening are collapsed.

- Identical workspace-open requests within 1.5 seconds are reused.
- Loadout and Raid Planner share one refresh key and one in-progress request.
- Repeated Assistant feed reads within two seconds reuse the current prepared feed.
- Manual Refresh bypasses these short reuse windows.

### Semantic manual refresh

The top Refresh button still asks the server to rescan current profile and market sources. It no longer blindly invalidates a materialized workspace after the source scan.

- Changed domains are invalidated by server change tracking.
- Unchanged summaries remain materialized and return immediately.
- Assistant Refresh rebuilds only source workspaces whose domains actually changed.
- A failed revision response falls back conservatively to refreshing the required Assistant sources.

### First-load HERMES tab state

The HERMES tab remains after Prestige, but its first-load visual state is now normalized after EFT finishes the native tab layout.

For the first few frames after the tab is attached, HERMES forces:

- The currently selected native tab, normally Gear, to remain selected.
- HERMES to remain visible but inactive.
- No ongoing per-frame placement or geometry copying.

This specifically addresses the case where the first Character-screen load looks wrong but becomes correct after selecting HERMES and then another native tab.

## Request lifecycle

### Startup and post-raid

1. Recheck semantic source revisions.
2. Warm Hideout, Crafts, Stash, and Loadout once.
3. Prepare the complete server-owned Assistant feed.
4. Recover any locally timed-out response from the now-materialized server cache.
5. Apply one complete client snapshot.

### Open a workspace

The selected workspace reads its prepared server response. Duplicate lifecycle callbacks during the same opening are coalesced.

### Press Refresh

1. `/hermes/recheck`
2. `/hermes/changes/<revision>`
3. Rebuild only source domains reported as changed.
4. Return the selected workspace or rebuilt Assistant feed.

## Build and installation

1. Overlay this source drop-in onto the current HERMES source tree.
2. Delete old `bin` and `obj` directories.
3. Open `HERMES.sln` in Visual Studio.
4. Confirm `SptRoot` points to the SPT 4.0.13 installation, normally `C:\RealSPT`.
5. Build the solution or `Hermes.Build`.
6. Restart both SPT Server and Escape from Tarkov.

The build project packages `HERMES-0.1.0-alpha14.0.8.zip`.
